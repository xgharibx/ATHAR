# Checklist الإطلاق للموبايل (Android via Capacitor)

## 1) تحقق من التطبيق الويب أولاً

- `npm install`
- `npm run verify`

> لا تنتقل لخطوات Android قبل نجاح التحقق الويب.

## 2) أنشئ منصة Android مرة واحدة

إذا لم يكن مجلد `android/` موجودًا:

```bash
npm run android:add
```

## 3) حدّث ملفات الويب داخل المشروع المحمول

```bash
npm run build
npm run android:sync
```

## 4) افتح Android Studio

```bash
npm run android:open
```

أو نفّذ المسار الكامل دفعة واحدة:

```bash
npm run android
```

## 5) فحوصات أساسية قبل التصدير

- افتح الشاشة الرئيسية وتأكد أن الشريط السفلي وزر التسبيح لا يتداخلان.
- اختبر صفحة القرآن على الهاتف:
  - التمرير
  - البحث
  - الإشارات المرجعية
  - وضع الشاشة الكاملة
- اختبر الإعدادات:
  - النسخ الاحتياطي
  - الاستعادة
  - الثيمات
- اختبر العمل بدون اتصال إن كان ذلك مطلوبًا.
- اختبر الإشعارات المحلية على جهاز فعلي إذا كانت مستخدمة.

## 6) أصول وهوية التطبيق

- راجع `appId` و `appName` في `capacitor.config.ts`.
- تأكد من أيقونات التطبيق داخل مشروع Android بعد الإنشاء.
- حدّث شاشة الإطلاق والألوان من Android Studio إذا لزم.

## 7) بناء APK / AAB

من Android Studio:

- `Build > Generate Signed Bundle / APK`
- اختر `Android App Bundle (AAB)` للنشر على المتجر
- أو `APK` للاختبار اليدوي

## 8) قبل الرفع النهائي

- جرّب نسخة release على جهاز فعلي واحد على الأقل.
- تأكد من سلامة RTL والخط العربي.
- تأكد أن أحجام اللمس مريحة في الصفحات الأساسية: الرئيسية، القرآن، البحث، الإعدادات.
