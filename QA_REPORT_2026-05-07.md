## Fixes Applied

| Area | Before | Fix | After |
|---|---|---|---|
| Production deep links | Direct routes like `/c/morning`, `/mushaf/1`, and `/hadith/bukhari` loaded `c/assets/*`, `mushaf/assets/*`, or `hadith/assets/*` and produced asset 404s. | Changed Vite base/PWA URLs to root paths in `vite.config.ts` and `vite.config.js`. | Built pages now load `/assets/*`; direct route reloads pass in production preview. |
| First-run tap handling | The startup loader could become invisible but still sit above the app for up to 6 seconds, intercepting onboarding taps. | Updated `index.html` loader script to immediately hide when React has already painted and set `pointerEvents = none`. | Real browser clicks now advance onboarding permission steps immediately. |
| Android install readiness | Latest tested web fixes were not in a fresh APK/AAB and version was still `1.2.6` / code `18`. | Synced Capacitor Android and bumped Android to `1.2.7` / code `19`. | Fresh Android release APK/AAB built and release metadata refreshed. |
